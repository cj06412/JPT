# 微信聊天记录 → JPT 人格抽取指南（开发者手工，送出前做）

> 配套：spec §2.2 / CLAUDE.md §3。零代码改动 —— 只编辑
> %APPDATA%\jpt\workdir\.claude\skills\JPT-wechat\SKILL.md（首启已生成模板）。
> 基础人格在 workdir\CLAUDE.md（settings 窗可编辑）；这里只做「风味增强」。

## 为什么分两层
- CLAUDE.md = always-on 基础人格（关系、风格、擅长、边界）—— 每轮都注入
- SKILL.md = 按需触发的语气增强 —— Claude 检测到「在跟小屿聊天」相关上下文才激活
  好处：基础人格稳定，语气样本不污染每一轮 token，且可单独迭代

## 步骤

1. **导出微信记录**：微信 PC 版 → 备份聊天记录，或用开源工具
   (WeChatMsg / PyWxDump 等) 导出和小屿的对话为文本。**只导你俩的**。

2. **脱敏**：删手机号 / 地址 / 第三方隐私 / 真名（如果不想出现）。
   保留语气、梗、节奏。

3. **三类提炼**，填进 SKILL.md 对应小节：
   - **口头禅 / 高频说法**：你回她时反复出现的短语、语气词、句尾习惯
   - **内部梗 / 共同回忆**：只有你俩懂的梗，每条配一句话上下文（让 Claude 用得对）
   - **语气样本**：5-10 条真实片段（去隐私），体现温度和节奏，不要太长

4. **写法要点**：
   - 不要写成「JPT 应该温柔」这种抽象指令（那是 CLAUDE.md 的事）
   - 要写成「具体怎么说」：原话、口癖、什么场合说什么
   - description frontmatter 要准确（决定 Claude 何时激活这个 skill）

5. **测试**：跑 dev / 装好的 app，跟 JPT 聊几句，看语气是否对味。
   不对就回来调 SKILL.md（改完重开对话框或 /clear 让会话刷新）。

## 注意
- SKILL.md 文件本身**不要进 git 仓库**（含私人语气样本）。它住在
  %APPDATA%\jpt\workdir\.claude\skills\JPT-wechat\，不在仓库内，天然隔离。
- references/ 子目录可放更长的历史话题摘录，SKILL.md 里引用文件名即可。
- 送出的安装包**不含**真实 SKILL.md（首启只生成占位模板）；真实内容你在
  小屿电脑上 / 送出前手工填，或做成「预填好 workdir」的方式随包带（看你怎么交付）。
